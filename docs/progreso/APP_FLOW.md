# APP_FLOW

## Flujo actual del MVP

### Paso 1. Conexión
La constructora conecta su wallet.

### Paso 2. Compra
La constructora compra una cantidad de caliche representada digitalmente.

### Paso 3. Balance
La app muestra el saldo adquirido y el inventario disponible.

### Paso 4. Redención
La constructora redime una parte del material.

### Paso 5. Burn
La redención ejecuta un burn que representa material consumido.

### Paso 6. Estado actualizado
La interfaz actualiza:
- inventario restante
- tokens quemados
- balance restante del usuario

## Flujo futuro operativo
1. La constructora compra derecho de extracción.
2. Se genera un permiso para el operador/chofer.
3. El operador llega al sitio.
4. El material se carga.
5. La báscula reporta salida.
6. El oráculo dispara la redención y burn.
7. El sistema deja registro verificable del consumo.
